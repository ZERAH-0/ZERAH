🦖 Code:Black — SDL2 기반 2D 액션 어드벤처 게임

  기간: 2025.03 - 2025.08
  참여: 개발 3인
  역할: 메인 프로그래머 · 보스전 시스템 총괄
  엔진/플랫폼: Custom(C++/SDL2) · PC



  📌 프로젝트 개요 (Overview)
  상용 엔진 없이 C++/SDL2로 렌더링·오디오·입력·충돌·상태 전이까지 직접 구현한 2D 액션 어드벤처. 인트로→스테이지1~7→엔딩으로 이어지는 페이즈 구조를 갖고, 보스전(Stage 6)을 핵심 전투 경험으로 설계. 네가 보스전 AI/패턴·UI·초기화 흐름을 중심으로 전체 전투 사이클을 구현했고, 중간/최종 발표도 담당.



  📌 구현 방식 (Development)
  • 프레임워크: SDL_Init/IMG/TTF/Mixer 초기화 → SDL_CreateWindow(800×800) · SDL_CreateRenderer(SDL_RENDERER_ACCELERATED) 메인 루프는 **약 30FPS 고정(33ms 간격)**으로 이벤트/업데이트/렌더 순환.
  • 페이즈 매니저: main_cbgame.cpp에서 game_phases[]로 각 씬 등록, PHASE_STAGE1~7/INTRO/ENDING/DIE 전환.
  • 스테이지 구조: 각 스테이지는 독립 클래스(stage1_home.cpp … stage7_she.cpp). 보스전은 stage6_bridge.(h|cpp)로 분리.
  • 자원 관리: IMG_Load→SDL_CreateTextureFromSurface로 배경/문/미션/HP/탄막 등 텍스처 로딩, SDL_mixer로 BGM(bgm_stage_boss.mp3)·SFX(피격/잠수/비늘 등) 볼륨/채널 관리.



  📌 팀 협업 & 기여 (Role & Contribution)
  • 보스전 전반(패턴/타이밍/HP UI/문 오픈/미션 연출/사운드/리셋 흐름) 직접 설계·구현.
  • 오브젝트 풀(비늘/총알)과 타이머 기반 상태기계(잠수/예고/공격)로 프레임 안정성 확보.
  • 스테이지 클래스 분리와 페이즈 전이 규약 정리로 병렬 개발 가능하도록 구조화.
  • 중간/최종 발표에서 구조·패턴·디버깅 포인트 설명 및 데모 진행.



  📌 주요 특징 (Features)
  • 페이즈 아키텍처: PhaseInterface{HandleEvents/Update/Render} 표준화 → 씬 독립성/가독성↑
  • 보스전 루프: 예고→잠수→공격→후퇴 4단계 구조, SFX·UI·충돌 동기화
  • UI/연출: HP 15칸, 미션 페이드, 미니맵/지도, 클리어 문 오픈.
  • 오디오: BGM/효과음 볼륨/채널 관리, 이벤트 지점에 정밀 트리거
  • 성능/안정성: 30FPS 고정 루프, 오브젝트 풀로 할당/해제 최소화, 재시작 완전 초기화



  📌 성과 & 인사이트 (Outcome & Insights)
  • 엔진 없이 저수준 API만으로 전투 사이클과 보스 AI를 완주하며 렌더/입력/오디오/상태 전이 전 과정 숙달.
  • 타이밍 제어·상태기계·오브젝트 풀링이 실전 안정성에 결정적임을 체득.
  • 팀 3인 체제에서 클래스 분할·규약화로 협업 속도와 유지보수성 향상.
  • 이후 과제(Projectile/Snake 등)에서도 패턴·초기화·UI 연동 설계가 재사용되는 기반이 됨.



 📌 핵심 기술 (Core Stack)

   C++ / SDL2 / Game Loop / State Machine / Boss AI / Object Pooling / SDL_mixer / IMG / TTF / 2D Rendering
